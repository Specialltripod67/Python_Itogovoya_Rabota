# Автоматизированный мониторинг и реагирование на угрозы

## Описание проекта

Данный проект представляет собой простой инструмент на Python для автоматизированного мониторинга и базового реагирования на угрозы информационной безопасности.

Скрипт выполняет следующие задачи:

- получает данные из двух источников:
  - логов Suricata (`eve.json`);
  - Vulners API;
- анализирует события безопасности;
- выявляет потенциальные угрозы:
  - подозрительные IP-адреса;
  - события с повышенной критичностью;
  - уязвимости с высоким уровнем CVSS;
- имитирует реагирование на угрозы через вывод сообщений о блокировке IP-адресов;
- сохраняет результаты анализа в отчёт;
- строит график по результатам анализа.

## Используемые источники данных

В проекте используются два источника данных:

1. **Логи Suricata**
   - файл `eve.json`;
   - используется для анализа сетевых событий безопасности.

2. **Vulners API**
   - используется для получения информации об уязвимостях;
   - в проекте выполняется запрос по идентификаторам CVE.

## Реализованный функционал

### 1. Загрузка и анализ логов Suricata
Скрипт:
- загружает JSON-файл `eve.json`;
- преобразует данные в `pandas.DataFrame`;
- выделяет только нужные поля:
  - `timestamp`
  - `event_type`
  - `src_ip`
  - `dest_ip`
  - `alert.signature`
  - `alert.severity`
- фильтрует только события типа `alert`.

### 2. Анализ угроз по Suricata
Скрипт:
- считает количество alert-событий;
- определяет топ IP-адресов по числу alert-срабатываний;
- определяет топ сигнатур атак;
- выделяет события с повышенной критичностью (`severity <= 2`);
- определяет подозрительные IP-адреса по правилу:
  - IP считается подозрительным, если по нему найдено **5 и более alert-событий**.

### 3. Реагирование на угрозы
При обнаружении подозрительных IP-адресов скрипт выводит в консоль сообщения вида:

`Simulated block for IP: <IP>`

Это является имитацией реагирования на угрозу.

### 4. Работа с Vulners API
Скрипт:
- выполняет запрос к Vulners API;
- получает данные по CVE;
- извлекает:
  - идентификатор уязвимости;
  - тип документа;
  - заголовок;
  - CVSS;
- помечает уязвимости с высоким баллом CVSS как требующие уведомления.

### 5. Формирование отчёта
По результатам анализа формируется файл:

- `output/report.csv`

В отчёт включаются:
- подозрительные IP-адреса из Suricata;
- события с высокой критичностью;
- уязвимости из Vulners API.

### 6. Построение графика
Скрипт строит график:

- **Top-5 source IP by alert count**

Результат сохраняется в файл:

- `output/chart.png`
